# 삼국지 카드 대전 — 구현 명세 v1

> GDD v1.1의 "열린 결정 사항"과 미정의 영역을 **성공 확률이 가장 높은 기본값**으로 채운 문서.
> 여기 적힌 모든 수치는 §9 튜닝 상수표에 모여 있으며, 플레이테스트 후 숫자만 교체하면 된다.
> 결정의 근거는 각 항목에 한 줄로 명시. GDD의 철학(운은 배치·구성에서, 항상형은 작게 조건부형은 크게)을 위반하는 값은 없음.

## 0. 결정 요약표 (GDD §10 → 확정 기본값)

| # | 열린 결정 | 채택안 | 근거 |
|---|---|---|---|
| 1 | 4수치 | **전투·정치·지략·통솔** | 이미 제작된 카드 아트(preview)가 이 4수치를 사용 중 → 아트 재작업 없음 |

## 1. 코어 턴 루프 (신규 정의 — GDD 최대 공백)

### 1.1 대전 구조
```
덱 30장 → 손패 5장으로 시작
매 턴: 양측이 손패에서 1장 비공개 선택 → 동시 공개 → 유효 전투력 비교
      → 높은 쪽 1점 획득, 낸 카드는 양쪽 모두 소모(무덤행)
      → 덱에서 1장 드로우해 손패 5장 유지
30턴 종료 시 다점자 승리. 16점 선취 시 조기 종료.
```

- **동시 공개(가위바위보형)** 채택 이유: 교대 방식은 후공이 항상 유리해 밸런스가 꼬임. 동시 공개는 "상대가 뭘 낼까"라는 심리전을 매 턴 만든다.
- **손패 5장** 채택 이유: 배치를 사전에 전부 고정하면 정보 공개 곡선(§2.9)이 무의미해짐. 정보가 열릴 때 대응할 수단(손패)이 있어야 정보전이 성립.
- **동률 처리:** 해당 턴 무승부(양쪽 0점). 30턴 후 동점이면 누적 유효 전투력 총합으로 판정, 그것도 같으면 무승부.
- **덱 구성:** 컬렉션에서 30장 선택. MVP에서 보유량이 30장 이하면 전체가 덱.

### 1.2 유효 전투력 공식 (확정 계수)
```
전투 가중치 = 전투×0.5 + 통솔×0.3 + 지략×0.2      ← 정치는 대전 판정에서 제외
유효 전투력 = 전투 가중치
            × 역사 배율 (§3.1)
            × 홈 배율 (§3.2)
            + 도시 이벤트 보정 (§4 도시별 규칙)
            + 아이템 보정 (MVP 제외, 2차에서 +0~15 가산)
```
- 정치를 판정에서 뺀 이유: 정치는 모사 보정(2:2)·도시 경제·미니게임의 핵심 수치로 역할 분리 → "전투형 vs 내정형" 카드 정체성이 생김. 조조(정치 94)는 대전보다 경영에서 빛나는 카드.

### 1.3 2:2 대결 (2차 — 규칙만 미리 확정)
- 별도 모드가 아니라 **한 턴에 2장(장수+모사)을 태우는 선택**. 상대가 1장이어도 성립.
- `2:2 유효 전투력 = 장수 유효 전투력 + (모사 지략×0.3 + 모사 정치×0.2) × 역사·홈 배율`
- 이겨도 똑같이 1점. 카드 2장 소모 = "반드시 이겨야 할 턴"에 거는 베팅 → 자원 심리전.
- 카드 타입: 전투 75 이상 = 장수, 지략 85 이상 = 모사 (중복 가능, 둘 다 아니면 장수 취급).

## 2. 수치 체계

### 2.1 4수치와 등급
- 4수치: **전투 / 정치 / 지략 / 통솔** (각 1~99)
- 장수마다 고정된 **원형 수치**(§5 로스터)가 있고, 카드 생성 시 등급 보너스가 랜덤 배분된다.

| 등급 | 보너스 포인트 | 특수 속성 |
|---|---|---|
| 1 | +0 | 없음 |
| 2 | +12 | 없음 |
| 3 | +28 | 1개 |
| 4 | +48 | 2개 |

### 2.2 랜덤 가중 배분 규칙
- 보너스 포인트를 4수치에 랜덤 분배하되, **한 수치에 전체의 60% 초과 배분 금지**, 각 수치 상한 99 클램프.
- → 같은 등급 3 관우라도 어떤 개체는 전투 몰빵(딜러형), 어떤 개체는 고른 분배(밸런스형). 거래 시장 가격 차이의 근원.

## 3. 판정 상수

### 3.1 역사 배율 (시대 적합도)
발동된 역사 시나리오의 연도 Y와 장수의 활동기·전성기(§5 로스터)를 비교:

| 조건 | 배율 |
|---|---|
| Y가 전성기 구간 안 | ×1.2 |
| Y가 활동기 구간 안 (전성기 밖) | ×1.0 |
| Y가 활동기 밖 (미등장/사후) | ×0.6 |
| 저항 속성 보유 시 하한 | ×0.8 보장 |

### 3.2 홈 배율 (연고/관리 도시)
| 상황 | 배율 |
|---|---|
| 풀 홈 (내 관리 도시) | ×1.10 + 발전도 보너스(최대 ×1.05 추가) |
| 하프 홈 (연고지, 미관리) | ×1.05 |
| 중립 | ×1.00 |
| 원정 (상대 관리 도시) | ×1.00 (상대가 풀 홈 이득) |

- MVP는 도시 관리가 없으므로 **연고지 하프 홈(×1.05)만 적용** — 장수의 출신·주활동 도시는 로스터의 연고 열 참조.

### 3.3 정보 공개 스케줄 (GDD §2.9 구체화)
| 시점 | 공개 |
|---|---|
| 대전 시작 | 역사 시나리오 + 룰, 전투 도시 + 상성, 국면 전환 횟수, 양측 국가 구성 **비율** |
| 5턴마다 (5,10,15,20,25) | 상대 잔여 덱 총 유효 전투력 갱신 |
| 10턴 | 상대 국가 구성 상세 (국가별 장수 수) |
| 20턴 | 상대 잔여 카드의 장수/모사 구분 공개 |

## 5. AI 모드 (MVP 상대)

- AI 덱: 로스터 30장 전체에서 랜덤 30장(중복 없음, MVP는 풀 자체가 30장이므로 동일 덱 미러전).
- AI 행동: 3단계 난이도.
  - **쉬움:** 손패에서 랜덤 선택.
  - **보통:** 자기 손패 중 기대 유효 전투력 최고 카드 (탐욕적).
  - **어려움:** 공개 정보(내 잔여 총 전투력·구성 비율) 기반 확률 추정으로 "이길 만한 턴엔 강카드, 질 것 같은 턴엔 최약체 버리기" (미니맥스 1수).
- 어려움 AI가 정보를 활용하는 모습 자체가 플레이어에게 정보전 교본이 된다.

## 6. 승패 보상 · 강화 (MVP)

- 승리: 신규 카드 5장 / 패배: 2장 (3연패 시 다음 패배 4장).
- 카드 배정: 로스터에서 랜덤, 등급 확률 — 1등급 60% / 2등급 27% / 3등급 10% / 4등급 3%.
- 강화: 같은 장수 + 같은 등급 2장 합성 → 등급 +1 (보너스 포인트 재추첨). 4등급은 강화 불가.
- 강화 시 소모된 개체의 ID·이력은 결과 카드의 `history`에 병합 기록 (§8 스키마, 거래 대비).

## 7. 2차 시스템 기본값 (구현은 나중, 규칙은 지금 확정)

### 7.1 국면 전환
- 게임 생성 시 0~3회 선택·공개. 전환 시점: 30턴을 (횟수+1)등분한 경계 턴 (예: 1회면 16턴째).
- 전환 시 역사 시나리오·도시 재추첨. 크리티컬 완충: 전환 후 역사 배율 하한 전 카드 ×0.7 (저항 카드는 ×0.8 유지).

### 7.2 일기토
- 트리거: 라이벌 쌍(여포↔관우, 여포↔장비, 마초↔허저, 장료↔태사자) 확정 / 그 외 양측 전투 85 이상.
- 진행: 5합. 매 합 기본 승자는 전투 우위자, 합마다 이벤트 추첨 — 허점 노출 10%(지략 우위자에게 발동, 그 합 역전), 필사의 일격 5%, 낙마 3%. 3합 선취 승.
- 반전 확률 = 기본(수치 차이 함수) × 역사 가중 × 도시 가중 × 국가 조합 가중 → **clamp(5%, 35%)**.
- 일기토 승자는 그 턴 2점 (일반 턴의 2배) — 유도/회피할 전략적 이유 부여.
- 종료 후 합별 로그를 영상 카드 데이터로 저장 (§8).

### 7.3 PvP
- 성장: 총량 고정 + 수평 확장 (스탯 재배분·상성 교체만 유료).
- 매칭: 승점 기반 단순 MMR. 승 5장/패 2장 유지하되 MMR 차이가 크면 약자 보상 +1장.

### 7.4 특수 속성 카탈로그 (등급 3=1개, 등급 4=2개, 랜덤 부여)
| 속성 | 유형 | 효과 |
|---|---|---|
| 위엄 | 항상형 | 아군 전체 유효 전투력 +1.5% |
| 명성 | 항상형 | 대전 보상 골드 +5% (전투 무관) |
| 수성 | 조건부 | 풀 홈 전장에서 +12% |
| 원정왕 | 조건부 | 상대 풀 홈 전장에서 상대 홈 배율 무효 |
| 철벽 | 조건부 | 역사 배율 하한 ×0.8 (저항 속성) |
| 설전 | 조건부 | 상대가 2:2로 낼 때 모사 보정 50% 감쇄 |
| 연환 | 조건부 | 직전 턴 아군 동일 국가 카드 승리 시 +10% |
| 복수 | 조건부 | 직전 턴 패배 시 +15% |

### 7.5 미니게임 (클래식 SLG — 내정·등용 중심)
- 연 4턴(분기). 커맨드: 개발 / 상업 / 치안 / 수색 / 등용 / 훈련 중 분기당 2개.
- 전쟁은 연 1회 "침공 이벤트"로 간소화 — 도시 방어력 + 주둔 장수 통솔 합산 자동 판정.
- 대전 연결: 등용 성공 장수 → 해당 장수 카드 1장 획득 / 내정 골드 → 대전 골드 (10:1).

## 8. 데이터 스키마 (Firestore, 거래 대비 — GDD §5.2 반영)

```
users/{uid}
  nickname, createdAt, gold, myCharacter{...}, lossStreak

cards/{cardId}                    ← 카드 "개체" 고유 문서 (장수 종류 아님)
  ownerId, generalId, grade
  stats: {combat, politics, intellect, leadership}   ← 랜덤가중 반영된 최종값
  traits: [{id, type: "always"|"conditional", value}] ← 정형화된 특수 속성
  history: [{event: "drop"|"enhance"|"trade", at, from, detail}]
  createdAt

matches/{matchId}
  players: [uidA, uidB|"AI"], scenario: {year, rules}, city, phaseShifts
  turn, scores, state: "placing"|"done"
  hands: {uidA: [...], uidB: [...]}   ← 보안 규칙으로 상대 손패 읽기 차단
  log: [{turn, cards, powers, winner, duel?: {...합별 로그}}]

duelReplays/{replayId}            ← 영상 카드 = 재생 가능한 로그 데이터
  matchId, turn, generals, rounds: [...], rarity
```

- PvP 시 손패 은닉은 Firestore 보안 규칙 + Cloud Function 판정으로 처리 (클라이언트 판정 금지 — 치팅 방지). MVP(AI전)는 클라이언트 판정으로 충분.
